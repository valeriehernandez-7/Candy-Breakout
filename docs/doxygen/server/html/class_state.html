◆ states

enum State::states

Game screens (states).

Enumerator
initial 

Refers to the InitialState, this is the welcome screen where the connection with the player is expected to access the loading screen.

play 

Refers to the PlayState, this is the game screen where the player must lose all lives to access the game over screen or must win all 10 levels to access the victory screen.

load 

Refers to the LoadState, this is the loading screen where the counter is expected to finish to access the game screen or to the next game level screen.

final 

Refers to the OverState, this is the last screen where the game score and result are shown to the player, the player decides whether to finish the game or play again.
